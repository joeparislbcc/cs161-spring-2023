Dictionaries

Dictionaries are similar to lists in that they are a collection of objects.

Like lists, dictionaries are mutable, can grow and shrink as needed, and can be nested (a dictionary can contain other dictionaries. As of Python 3.7, Dict order is guaranteed to be insertion order.

Unlike lists, whose elements are accessed via an index, the items in a dictionary are accessed via a key Prior .

You access values in a dictionary via its key:

students = dict(
		X00335975 = 'Stephanie Henderson',
		X00352898 = 'Natalie Hawkins',
		X00354152 = 'Aaron Hines',
		X00665112 = 'Susan Reed',
		X00726007 = 'Brian Jackson'
)

print(students['X00335975']

Restrictions on Dictionary Keys

  • A given key can only appear in a dictionary once.
  • Dictionary keys must be of an immutable type such as integer, float, string, Boolean, tuple, etc.

Restrictions on Dictionary Values

There are no restrictions on dictionary values.

A quick review of the data types you've seen so far...

Lists are mutable sequences of elements, best suited for situations where you need to store and manipulate a collection of homogeneous elements in a specific order.

They are commonly used for tasks such as building up sequences of values, representing matrices and other multidimensional arrays, and implementing stacks, queues, and other data structures.

Tuples are immutable sequences of elements, best suited for situations where you need to store a collection of heterogeneous elements that won't change and whose order is important.

They are commonly used for tasks such as returning multiple values from a function, unpacking sequences into variables, and representing fixed-length records.

Dictionaries are mutable, unordered collections of key-value pairs, best suited for situations where you need to store and retrieve values based on a key of some sort.

They are commonly used for tasks such as caching results, representing data with a hierarchical structure, and implementing lookup tables.

Sets

Sets are mutable, unordered collections of unique elements, best suited for situations where you need to perform set operations such as union, intersection, and difference.

They are commonly used for tasks such as deduplicating data, finding common elements between multiple collections, and testing for membership.
