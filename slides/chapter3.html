Chapter 3

Algorithms and Program Development

Informally, an algorithm is any well-defined procedure that takes some value, or set of values, as input and produces some value, or set of values, as output in a finite amount of time.

An algorithm is thus a sequence of computational steps that transform the input into the output.

Definition of the sorting problem:

Input: A sequence of n numbers ⟨a1, a2, ...., an⟩.
Output: A permutation (reordering) ⟨a′1, a′2, ...., a′n⟩ of the input sequence such that a′1 ≤ a′2 ≤ ... ≤ a′n.
Suppose you need to sort a sequence of numbers into monotonically increasing order. this problem arises frequently in practice. Here we formally define the sorting problem.

Most often we are concerned about the efficiency of algorithms and our usual measure of efficiency is speed: how long does an algorithm take to produce its result?

There are some problems, however, for which we know of no algorithm that runs in a reasonable amount of time. These are known as NP-complete problems.
